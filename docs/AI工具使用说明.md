# AI 工具使用说明

## 一、使用的 AI 工具

### 1.1 主要工具
| 工具 | 用途 | 使用方式 |
|------|------|---------|
| **Cursor / Trae IDE** | 代码生成、重构、调试 | 自然语言描述需求，生成代码框架 |
| **ChatGPT / Claude** | 方案设计、问题排查 | 讨论架构设计、分析错误日志 |
| **通义灵码** | 代码补全、注释生成 | 日常编码辅助 |

### 1.2 使用原则
1. **AI 生成框架，人工审核细节**：AI 生成的代码必须经过人工审查，确保逻辑正确
2. **关键逻辑手写**：核心算法（如断行合并、置信度分级）手写实现，AI 仅辅助优化
3. **测试用例人工设计**：测试问题基于文档内容人工设计，确保覆盖关键场景
4. **错误日志人工分析**：AI 辅助定位问题，但最终决策由人工做出

## 二、AI 辅助开发流程

### 2.1 代码生成流程
```
需求描述 → AI 生成代码框架 → 人工审核逻辑 → 测试验证 → 修正优化
```

**示例：OCR 后处理模块开发**
1. 向 AI 描述需求："实现 OCR 文本后处理，包括断行合并、纠错、置信度分级"
2. AI 生成基础代码框架
3. 人工审核：
   - 断行合并规则是否合理？
   - 纠错字典是否覆盖常见错误？
   - 置信度分级阈值是否合适？
4. 编写单元测试验证
5. 根据测试结果修正逻辑

### 2.2 问题排查流程
```
错误日志 → AI 分析可能原因 → 人工验证假设 → 修复代码 → 回归测试
```

**示例：向量库初始化错误**
1. 错误信息：`TypeError: VectorStore.__init__() got an unexpected keyword argument 'collection_name'`
2. AI 分析：可能是参数名错误
3. 人工验证：查看 VectorStore 类定义，确认参数名为 `index_name`
4. 修复代码：修改参数名
5. 回归测试：重新运行构建流程

## 三、AI 输出校验方法

### 3.1 代码校验
- **静态检查**：使用 `flake8`、`mypy` 检查代码质量
- **单元测试**：每个模块编写独立测试用例
- **集成测试**：端到端测试完整流程

### 3.2 逻辑校验
- **边界条件**：测试空输入、异常输入、极端值
- **兜底逻辑**：确保异常场景不会导致程序崩溃
- **结果验证**：人工核对 AI 生成的答案与文档原文

### 3.3 测试校验
- **测试用例覆盖**：正文、表格、无答案、模糊问题
- **回归测试**：每次修改后运行完整测试集
- **人工复核**：AI 生成的测试结果需人工确认

## 四、AI 使用记录

### 4.1 本次开发 AI 使用情况
| 模块 | AI 辅助内容 | 人工修改内容 |
|------|-----------|-------------|
| 文件转换器 | 生成 LibreOffice 转换逻辑 | 调整异常处理、日志记录 |
| PDF 判别器 | 生成文本量判断逻辑 | 优化阈值、增加调试信息 |
| 内容解析器 | 生成 PyMuPDF 提取逻辑 | 增加条款识别、表格检测 |
| OCR 后处理器 | 生成基础框架 | 手写断行合并、纠错逻辑 |
| 分块器 | 生成滑动窗口逻辑 | 调整窗口大小、重叠策略 |
| 向量库 | 生成 ChromaDB 操作 | 增加元数据字段、检索优化 |
| 答案生成器 | 生成 prompt 模板 | 优化溯源格式、风险提示 |
| 自检器 | 生成幻觉校验逻辑 | 调整判断阈值、边界处理 |

### 4.2 AI 辅助效率提升
- **代码生成效率**：提升约 40%，减少重复编码
- **问题排查效率**：提升约 50%，快速定位错误原因
- **文档编写效率**：提升约 60%，自动生成框架后人工完善

### 4.3 AI 局限性及应对
| 局限性 | 应对方法 |
|--------|---------|
| AI 生成的代码可能有逻辑错误 | 人工审核 + 单元测试 |
| AI 不理解业务场景 | 人工提供业务上下文 |
| AI 无法保证测试覆盖 | 人工设计测试用例 |
| AI 生成的答案可能幻觉 | 人工核对原文 + 自检机制 |

## 五、总结

AI 工具在本次开发中发挥了重要作用，主要体现在：
1. **快速生成代码框架**，减少重复劳动
2. **辅助问题排查**，提高调试效率
3. **生成文档模板**，规范项目结构

但 AI 无法替代人工的关键工作：
1. **核心逻辑设计**：需要深入理解业务场景
2. **测试用例设计**：需要人工判断覆盖范围
3. **结果验证**：需要人工核对准确性
4. **架构决策**：需要权衡取舍

**核心原则**：AI 是工具，不是决策者。开发者必须对最终结果负责。
