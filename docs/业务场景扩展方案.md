# 业务场景扩展方案

## 一、扩展设计原则

### 1.1 核心框架不变
- 7 个核心模块（转 PDF → 判别 → 解析 → 分块 → 检索 → 生成 → 自检）保持不变
- 仅通过配置层适配不同业务场景

### 1.2 配置驱动扩展
- 业务关键词库
- 纠错字典
- 分块策略参数
- Prompt 模板
- 阈值配置

### 1.3 插件化设计
- 新增业务适配器（继承基类）
- 替换业务配置
- 无需修改核心代码

## 二、金融场景适配

### 2.1 场景特点
- 大量数值表格（金额、利率、百分比）
- 专业术语多（如"年化收益率"、"夏普比率"）
- 合规要求高（数据准确性）

### 2.2 适配方案

#### 2.2.1 新增金融术语纠错字典
```python
# config/finance_correction.py
FINANCE_CORRECTION_DICT = {
    '年化收益': '年化收益率',
    '夏普比率': '夏普比率',
    '最大回撤': '最大回撤',
    '波动率': '波动率',
    'β系数': 'β系数',
    'α收益': 'α收益',
}
```

#### 2.2.2 优化数值表格解析
```python
# config/finance_config.py
FINANCE_TABLE_CONFIG = {
    'number_patterns': [
        r'\d+\.\d+%',      # 百分比
        r'¥?\d+,?\d*',     # 金额
        r'\d+\.\d+',       # 小数
    ],
    'validation_rules': {
        'percentage': (0, 100),   # 百分比范围
        'amount': (0, 1e12),      # 金额范围
    }
}
```

#### 2.2.3 调整分块策略
```python
# config/finance_config.py
FINANCE_CHUNK_CONFIG = {
    'chunk_size': 300,      # 金融文档段落较短
    'chunk_overlap': 50,    # 减少重叠
    'table_independent': True,  # 表格独立存储
}
```

### 2.3 风险提示策略
- 数值类答案必须标注来源表格页码
- 金额、利率等关键数据增加二次校验
- 低置信度内容自动标注"可能存在识别偏差"

## 三、合同场景适配

### 3.1 场景特点
- 条款层级复杂（章、节、条、款、项）
- 日期、金额、权责条款关键
- 合同主体识别重要

### 3.2 适配方案

#### 3.2.1 新增合同条款识别规则
```python
# config/contract_config.py
CONTRACT_CLAUSE_PATTERNS = [
    r'第[一二三四五六七八九十百千\d]+章',     # 章
    r'第[一二三四五六七八九十百千\d]+节',     # 节
    r'第[一二三四五六七八九十百千\d]+条',     # 条
    r'第[一二三四五六七八九十百千\d]+款',     # 款
    r'[（(]\d+[）)]',                        # 项
]
```

#### 3.2.2 新增合同实体识别
```python
# config/contract_config.py
CONTRACT_ENTITIES = {
    'party_a': r'甲方[：:]\s*([^\n]+)',
    'party_b': r'乙方[：:]\s*([^\n]+)',
    'date': r'(\d{4}年\d{1,2}月\d{1,2}日)',
    'amount': r'金额[：:]\s*([^\n]+)',
}
```

#### 3.2.3 优化条款层级识别
```python
# config/contract_config.py
CONTRACT_LEVEL_WEIGHTS = {
    '章': 1,
    '节': 2,
    '条': 3,
    '款': 4,
    '项': 5,
}
```

### 3.3 风险提示策略
- 日期、金额类答案必须标注来源
- 权责条款增加完整性检测
- 模糊条款自动标注"可能存在歧义"

## 四、合规文档适配

### 4.1 场景特点
- 法规引用多（如"根据《XX 法》第 X 条"）
- 合规要求严格
- 更新频繁

### 4.2 适配方案

#### 4.2.1 新增法规引用识别
```python
# config/compliance_config.py
LAW_REFERENCE_PATTERNS = [
    r'《[^》]+》第[一二三四五六七八九十百千\d]+条',
    r'根据[^，,。；;]+规定',
    r'参照[^，,。；;]+执行',
]
```

#### 4.2.2 新增合规要求提取
```python
# config/compliance_config.py
COMPLIANCE_KEYWORDS = [
    '应当', '必须', '不得', '禁止', '严禁',
    '可以', '允许', '建议', '推荐',
]
```

#### 4.2.3 调整分块策略
```python
# config/compliance_config.py
COMPLIANCE_CHUNK_CONFIG = {
    'chunk_size': 400,      # 合规文档段落较长
    'chunk_overlap': 80,    # 增加重叠保留上下文
    'clause_independent': True,  # 条款独立存储
}
```

### 4.3 风险提示策略
- 法规引用必须标注来源条款
- 合规要求增加完整性检测
- 过期法规自动标注"可能已更新"

## 六、扩展实施步骤

### 6.1 新增业务场景
1. 在 `config/` 目录下创建业务配置文件
2. 定义业务关键词、纠错字典、分块策略
3. 创建业务适配器（可选）
4. 更新 prompt 模板

### 6.2 配置示例
```python
# config/my_business_config.py
MY_BUSINESS_CONFIG = {
    'correction_dict': {...},
    'chunk_config': {...},
    'clause_patterns': [...],
    'risk_warning_rules': [...],
}
```

### 6.3 使用示例
```python
from main import create_rag_robot
from config.my_business_config import MY_BUSINESS_CONFIG

# 创建机器人（指定业务类型）
robot = create_rag_robot(
    collection_name="my_doc",
    business_type="my_business",
    config=MY_BUSINESS_CONFIG
)

# 构建知识库
robot.build("path/to/document.pdf")

# 问答
result = robot.ask("业务相关问题？")
```

## 七、扩展验证方法

## 八、总结

本系统通过**配置驱动 + 插件化设计**实现业务场景扩展：
1. **核心框架不变**：7 个模块保持稳定
2. **配置层适配**：通过配置文件调整业务逻辑
3. **插件化扩展**：新增业务适配器无需修改核心代码
4. **验证方法统一**：使用相同测试框架验证不同场景

这种设计确保系统能够快速适配金融、合同、合规、产品手册等不同业务场景，同时保持核心框架的稳定性和可维护性。
