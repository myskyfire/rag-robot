# 问题与解决方案汇总

## 1. Python版本问题
**问题**: Python 3.14太新，PaddlePaddle不支持，导致OCR无法安装
**解决方案**: 使用Python 3.12创建独立虚拟环境 `.venv312`，所有依赖（包括PaddleOCR）在该环境下正常运行
**注意**: 绝对不要降级requirements.txt中的依赖版本，除非确认无法兼容

## 2. 引用来源格式问题
**问题**: 引用来源只显示"第X页"，同一页面多次出现（如第3页出现6次），无法快速判断内容
**解决方案**: 修改 `answer_checker.py` 的 `format_result` 方法，引用来源现在显示：页码 + 内容类型 + 前100字符摘要
**修改文件**: `core/answer_checker.py`

## 3. 溯源信息未去重
**问题**: `_format_sources` 没有去重，每个检索到的chunk都作为独立来源列出
**解决方案**: 修改 `answer_generator.py` 的 `_format_sources` 方法，按页码去重，每页只保留一个来源
**修改文件**: `core/answer_generator.py`

## 4. 自检结果emoji编码问题
**问题**: `answer_checker.py` 中使用 `⚠` emoji字符，Windows控制台默认GBK编码无法显示，导致 `'gbk' codec can't encode character '\u26a0'` 错误
**解决方案**: 将 `⚠` 替换为 `[警告]`，兼容Windows控制台编码
**修改文件**: `core/answer_checker.py`

## 5. 虚拟环境混乱
**问题**: 终端默认使用 `.venv`（Python 3.14），但项目需要 `.venv312`（Python 3.12）
**解决方案**: 运行脚本时明确指定Python解释器路径：`.\.venv312\Scripts\python.exe script.py`
**注意**: 不要随意切换或重建虚拟环境，`.venv312` 已配置好所有依赖

## 6. OCR导出结果为空
**问题**: 使用Python 3.14环境运行OCR导出脚本，PaddleOCR未安装，所有页面跳过OCR识别
**解决方案**: 使用 `.venv312` 环境运行：`.\.venv312\Scripts\python.exe export_ocr_results.py`

## 7. DocumentPreprocessor初始化参数错误
**问题**: `export_ocr_results.py` 中 `DocumentPreprocessor()` 初始化缺少 `pdf_path` 参数
**解决方案**: 修改为 `DocumentPreprocessor(PDF_PATH)`，并调用 `process()` 而非 `preprocess()`
**修改文件**: `export_ocr_results.py`

## 8. 多模态模型兜底功能
**问题**: 需要实现OCR识别差时的多模态模型兜底方案
**解决方案**: 
- 新增 `core/multimodal_fallback.py` - 多模态兜底服务
- 新增 `docs/多模态兜底说明.md` - 详细说明文档
- 修改 `config/settings.py` - 添加多模态模型配置参数
- 修改 `core/content_parser.py` - 集成多模态兜底逻辑
- 配置可选，不配置不影响主流程
- 多层降级策略确保异常时不崩溃

## 9. requirements.txt版本问题
**问题**: 曾错误将 `paddlepaddle>=3.0.0` 降级为 `paddlepaddle>=2.6.0`
**解决方案**: 已恢复为 `paddlepaddle>=3.0.0`
**重要原则**: 不是没有一点办法绝对不要降版本

```bash
.\.venv312\Scripts\Activate.ps1
```
